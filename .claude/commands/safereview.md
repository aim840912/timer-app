# Safe Code Review

請在 **Plan Mode（唯讀）** 中審查以下程式碼：

$ARGUMENTS

## 審查檢查清單：

### 1. 程式碼品質
- [ ] 是否遵循專案的 coding style？
- [ ] 是否有適當的 TypeScript 類型定義？
- [ ] 是否有重複程式碼可以抽取？
- [ ] 函數和元件的複雜度是否合理？

### 2. 架構一致性
- [ ] 是否遵循專案現有的架構模式？
- [ ] API 路由是否使用統一的錯誤處理中間件？
- [ ] 是否使用專案的 logger 系統（apiLogger/dbLogger）？
- [ ] 是否遵循 CLAUDE.md 中的規範？

### 3. 安全性
- [ ] 是否有 SQL injection 風險？
- [ ] 是否有 XSS 漏洞？
- [ ] 敏感資料是否正確處理？
- [ ] 認證授權是否正確實作？

### 4. 效能
- [ ] 是否有 N+1 查詢問題？
- [ ] 是否有不必要的重新渲染？
- [ ] 是否正確使用快取？
- [ ] Bundle 大小影響是否可接受？

### 5. 測試
- [ ] 是否需要新增測試？
- [ ] 現有測試是否需要更新？
- [ ] 邊界情況是否被考慮？

## 輸出格式：

### ✅ 優點
[列出做得好的地方]

### ⚠️ 建議改進
[列出可以改進的地方，按優先級排序]

### 🔴 必須修正
[列出必須修正的問題]

### 📝 範例程式碼
[如果有建議，提供具體的改進程式碼範例]

**重要**：這是審查階段，請不要執行任何程式碼修改，只提供分析和建議。
